Prompting Twice might be the way

March 4, 2026

This comes from me randomly finding a reddit thread based on this paper done by the members of DeepMind, it got me curious about how useful this could actually be.

Essentially, the team at Google and came to the conclusion that: "When not using reasoning, repeating the input prompt improves performance for popular models (Gemini, GPT, Claude, and Deepseek) without increasing the number of generated tokens or latency."

It was found that the order of user tokens in the query can affect model performance with regard to outputs generated, i.e., <CONTEXT><QUESTION> performs differently from <QUESTION><CONTEXT>.

They proposed to repeat the prompt, i.e., input from <QUESTION> to <QUESTION><QUESTION>. This allows each prompt token to attend to every other prompt token. When not using reasoning, prompt repetition improves the performance of LLMs without increasing the length of the generated outputs or latency.

This was tested on 7 popular models of varying sizes: Gemini 2.0 Flash, Gemini 2.0 Flash Lite, GPT-4o-mini, GPT-4o, Claude 3 Haiku, Claude 3.7 Sonnet, and Deepseek V3 and was simultaneously tested on a set of 7 benchmarks in several configurations: ARC (Challenge), OpenBookQA, GSM8K, MMLU-Pro, MATH and 2 custom benchmarks called NameIndex and MiddleMatch.

NameIndex is essentially giving the model a list of N names and asking it to output the ith name on the list. We use N = 50, i = 25.

MiddleMatch is when the model gets a list of N names or numbers (out of a total possible set of K, i.e., K < N means there will be repeating elements), and is asked to output the name/number located directly between two given ones. We use N = 40, K = 10.

They compare prompt repetition to 2 additional variants namely Prompt Repetition (Verbose) and Prompt Repetition ×3. We observe that they perform similarly for most tasks and models, and sometimes outperform vanilla prompt repetition. Notably, Prompt Repetition ×3 often substantially outperforms vanilla prompt repetition.

To demonstrate that the gains are indeed due to repeating the prompt and not simply increasing the length of the input, they also evaluate the Padding method, which pads the inputs with periods (".") to the same length as prompt repetition and, as expected, does not improve performance.

The final conclusion they came to is that prompt repetition does not change the length or format of generated outputs, and it might be a good default for many models and tasks when reasoning is not used.

Usecases of this can vary but the one that immediately comes to mind is a semi-distillation method where we could only use prompt repetition at inference, get the improved answers and feed this to the finetuned model by single prompt + improved answer sets. In an ideal world, this would increase baseline accuracies of finetuned models but more research has to be done regarding this.
